    Background

    What is RGB-to-RAW Reconstruction?

    ISP Pipeline (Forward)

    RAW Sensor Data (14-bit, linear)
    ↓ Demosaic → White Balance → Tone Mapping → Gamma
    sRGB Image (8-bit, compressed)

    Inverse ISP (Our Task)

    sRGB Image
    ↓ Ill-posed inverse problem
    Reconstructed RAW Data

    Why RAW Matters

    • Higher Bit-Depth: RAW retains 12-14 bit linear sensor data vs 8-bit sRGB
    • Better for CV Tasks: Object detection, denoising, super-resolution all benefit
    • Rich Scene Information: Unprocessed data preserves more dynamic range

    Key Challenge

    • Ill-posed Nature: Multiple RAW images can produce the same RGB output
    • Information Loss: Quantization and clipping in ISP pipeline are irreversible
    • Cost: Collecting large-scale RAW datasets is expensive
    Motivation

    Limitations of Existing Approaches

    1

    ISP Prior-based Methods

    E.g., UPI, CycleISP, CycleR2R

    • Decompose ISP and invert each stage
    • Problem: Cumulative errors from stage-wise modeling
    • Problem: Require metadata (exposure, WB) often unavailable
    2

    End-to-End Regression Methods

    E.g., InvISP, ReRAW

    • Direct neural network mapping RGB→RAW
    • Problem: Detail inconsistency and color deviation
    • Problem: Poor generalization across cameras
    3

    Diffusion-based Methods

    E.g., DDPM, RAW-Diff

    • Generative approach with strong expressiveness
    • Problem: Stochastic noise causes structural distortions
    • Problem: Hundreds of denoising steps → very slow
    ★

    RAW-Flow (This Work)

    Deterministic Latent Flow Matching

    • Generative yet deterministic — no stochastic noise
    • Efficient — direct vector field supervision
    • Accurate — faithful detail & color reconstruction
    Contributions

    Three Key Contributions

    01

    Deterministic Latent Flow Matching (DLFM)

    First to reformulate RGB-to-RAW reconstruction as a deterministic latent transport problem. Unlike diffusion models that use stochastic noise, DLFM learns a deterministic vector field via flow matching to map RGB latent features to RAW latent features. A cross-scale context guidance module injects hierarchical RGB features to enhance luminance and detail fidelity.

    02

    Dual-domain Latent Autoencoder (DLAE)

    A novel autoencoder architecture that jointly encodes RGB and RAW images into their respective latent spaces. A dual-domain feature alignment constraint stabilizes RAW encoder training by aligning shallow features with the more robust RGB encoder. Cross-domain feature injection improves reconstruction fidelity.

    03

    State-of-the-Art Performance

    Extensive experiments on FiveK-Nikon, FiveK-Canon, and PASCALRAW benchmarks demonstrate that RAW-Flow outperforms all prior regression-based and diffusion-based methods both quantitatively (PSNR/SSIM) and qualitatively (visual comparison).

    Input RGB Image

    The framework takes an sRGB image as input. This is the readily available 8-bit image that has been processed through the camera's ISP pipeline. The goal is to reconstruct the original RAW sensor data from this compressed representation.

    RGB Encoder (ℰrgb)

    Part of the Dual-domain Latent Autoencoder (DLAE). Encodes the RGB image into a compact latent representation zrgb (= z0). This encoder is trained jointly with the RAW encoder, where shallow RGB features serve as alignment targets for the RAW encoder, promoting stable training.

    Deterministic Latent Flow Matching (DLFM)

    The core innovation. Learns a deterministic vector field v̂θ(zt, t) that transports the RGB latent z0 to the RAW latent z1 along a straight interpolation path. During inference, K=20 Euler integration steps are used. A cross-scale context guidance module injects hierarchical multi-scale RGB features into the flow estimation U-Net.

    RAW Decoder (𝒟raw)

    Decodes the transported latent representation ẑraw back to pixel space to produce the reconstructed RAW image. Receives cross-domain guidance from the RGB encoder's shallow features to reinforce structural detail and color consistency during decoding.

    Reconstructed RAW Image

    The final output: a high-fidelity RAW image that faithfully recovers structural details, global luminance, and color information from the input sRGB image. This can be used for downstream tasks like object detection, denoising, and super-resolution.

    Component 1

    Dual-domain Latent Autoencoder (DLAE)

    Design Motivation

    Training a RAW autoencoder alone is significantly less stable than an RGB autoencoder. Direct pixel-wise loss fails to produce perceptually accurate RAW outputs.

    Solution: Joint Training

    • Shared optimization: RGB and RAW autoencoders are jointly optimized with reconstruction loss (L2 + perceptual)
    • Feature alignment loss ℒfea: Aligns shallow features of the RAW encoder with the RGB encoder as a stable reference
    • Cross-domain injection: Shallow RGB encoder features are injected into RAW decoder layers for structural guidance

    Feature Alignment Loss

    ℒfea = Σl∈S ‖flraw − flrgb‖22

    S = set of shallow layers; aligns low-level features while preserving domain-specific deep features

    Key Insight

    Although RGB and RAW images differ in format and statistics, they capture the same underlying scene. Low-level features (edges, textures) should be similar, while deeper features encode domain-specific information and are left free to diverge.

    Component 2

    Deterministic Latent Flow Matching (DLFM)

    Latent Transport as ODE

    Instead of adding noise and denoising (diffusion), DLFM defines a straight interpolation path between RGB and RAW latent representations and learns the velocity field that moves along this path.

    zt = t · z1 + (1−t) · z0,   t ∈ [0, 1]

    z0 = ℰrgb(Irgb),   z1 = ℰraw(Iraw)

    Training Objective

    ℒflow = 𝔼t∼U(0,1) ‖v̂θ(zt, t) − v(zt, t)‖²

    Target velocity: v(zt, t) = z1 − z0 (constant direction)

    Advantage

    Direct supervision of the vector field → no stochastic sampling, simpler training, numerically stable, efficient inference

    Stochastic Flow Matching

    zt = t · z1 + (1−t) · ε,   ε ∼ N(0, I)
    • Initial state sampled from Gaussian noise
    • Suitable for unconditional generation
    • Introduces unnecessary uncertainty for deterministic mapping tasks

    Deterministic (DLFM - Ours)

    zt = t · z1 + (1−t) · z0
    • Initial state = actual RGB latent
    • Mapping is inherently one-to-one (fixed camera & scene)
    • More stable training & precise reconstruction

    Euler Integration (K=20 steps)

    zk+1 = zk + Δt · v̂θ(zk, tk),   z0 = zrgb,   Δt = 1/K
    z0 = zrgb
    → +Δt·v̂ →
    z1
    → +Δt·v̂ →
    z2
    → ... →
    zK-1
    → +Δt·v̂ →
    zK = ẑraw

    After K=20 steps, the final latent is decoded: Îraw = 𝒟raw(ẑraw)

    Cross-scale Context Guidance

    Relying solely on latent embeddings limits awareness of global luminance, color consistency, and fine local structures.

    • Residual blocks extract initial features from input RGB
    • U-Net encoder captures hierarchical features {firgb}i=1..L at multiple scales
    • Each scale feature provides spatially aligned guidance to the corresponding flow U-Net layer

    Conditioned Velocity Field

    v̂θ(zt, t | {firgb}i=1..L)

    Why Multi-scale?

    Global semantics (coarse scales) + local textures (fine scales) together enable precise alignment between RGB and RAW latent spaces. Single-scale guidance cannot capture both.

    Training

    Two-Stage Training Strategy

    Stage 1: Train DLAE (200 epochs)

    Train dual-domain autoencoders with paired RGB-RAW images.

    RGB Autoencoder Loss:

    ℒrecrgb = ‖Ĩrgb − Irgb‖² + λ₁·‖φ(Ĩrgb) − φ(Irgb)‖²

    RAW Autoencoder Loss:

    ℒraw = ℒrecraw + λ₂·ℒfea

    Stage 2: Train DLFM (100 epochs) → End-to-End Fine-tuning

    Freeze DLAE, train flow matching module, then fine-tune everything jointly.

    End-to-end Loss:

    ℒ = ‖Îraw − Iraw‖₁ + λ₃·‖φ(Îraw) − φ(Iraw)‖²

    Joint optimization alleviates representation mismatches between DLAE and DLFM.

    Conclusion

    Summary & Key Takeaways

    🔄

    New Paradigm

    First to formulate RGB-to-RAW as deterministic latent transport via flow matching

    🏗️

    Novel Architecture

    DLAE with feature alignment + DLFM with cross-scale context guidance

    🏆

    SOTA Results

    Best performance on all 3 benchmarks, surpassing both regression & diffusion methods

    Why It Works

    • Deterministic > Stochastic: RGB-to-RAW is a one-to-one mapping under fixed camera/scene — noise-based generation adds unnecessary uncertainty
    • Latent space is key: Operating in latent space enables more compact and meaningful representations than pixel-space methods
    • Dual-domain synergy: Leveraging the stability of RGB learning to bootstrap RAW encoder training is a clever and effective strategy
    • Multi-scale matters: Both global context and local details are critical for faithful RAW reconstruction
